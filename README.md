# 网络&Git一体化诊断修复工具

一款面向普通用户和开发人员的**一键式网络+Git专项诊断修复工具**，基于Python开发，跨平台、无门槛、轻量。

## 🎯 核心功能

- **零代码操作**: 一键检测，自动生成可视化故障报告
- **精准定位**: 覆盖代理失效、端口不通、Git配置错误、DNS正常但业务失败等全场景
- **智能修复**: 自动修复90%常见故障，无需手动输命令
- **专业诊断**: 网络测试工程师级别的检测逻辑，定位根因

## 📋 适用场景

1. 网络时好时坏，部分网站/API被限制
2. Git配置代理后失效、清除代理仍无法拉取代码
3. DNS解析异常、端口被屏蔽、系统代理冲突
4. 路由追踪超时、HTTPS/443端口连接失败
5. 无需专业网络知识，自动排查+修复

## 📊 检测项

### 基础网络检测
- 本机网卡状态、IP地址
- 网关连通性
- 公网连通性（223.5.5.5/8.8.8.8）

### DNS专项检测
- 目标域名解析（github.com等）
- 解析速度测试
- 公共DNS可用性

### 端口&代理检测
- 指定端口监听检测（10808/7890等代理端口）
- 目标端口连通性（github.com:443）
- 系统代理检测
- 代理有效性验证

### Git专项检测
- Git全局配置（http.proxy/https.proxy）
- Git本地仓库配置
- Git远程仓库连通性
- Git SSL验证状态

### 路由&SSL检测
- 路由追踪（tracert）
- HTTPS证书验证
- 防火墙出站规则

## 🔧 智能修复策略

| 检测到的故障 | 自动修复方案 | 优先级 |
|-------------|-------------|--------|
| Git配置了无效代理 | 一键清除Git全局/本地代理配置 | 最高 |
| github.com 443端口不通 | 临时禁用Git SSL验证 | 高 |
| DNS解析异常 | 自动刷新DNS缓存 | 高 |
| 系统代理与Git代理冲突 | 清除系统代理+Git代理 | 中 |
| Git配置错误 | 重置Git全局网络配置 | 中 |

## 🎨 故障分级

- **致命**: 无公网、网卡禁用 → 无法使用网络
- **严重**: Git端口不通、无效代理、DNS解析失败 → 核心业务不可用
- **警告**: DNS慢、路由超时、代理配置冗余 → 网络不稳定
- **正常**: 所有项通过

- **JSON**: 结构化数据，便于程序处理

## 🔍 操作流程

1. **启动工具** → 双击运行或 `python main.py`
2. **一键全量检测** → 自动执行所有检测项
3. **生成故障报告** → 彩色标注：正常/警告/严重/致命
4. **一键自动修复** → 工具自动执行修复命令
5. **修复后验证** → 重新检测，确认故障解决
6. **导出日志** → 保存报告用于复杂问题排查

## ⚠️ 注意事项

1. 部分修复操作需要管理员/root权限
2. 禁用Git SSL验证仅为临时解决方案，验证后建议重新启用
3. 定期更新工具以获取最新检测逻辑

3. Git配置是否正确
